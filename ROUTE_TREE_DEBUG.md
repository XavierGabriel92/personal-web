# TanStack Router - Route Tree Auto-Generation Debug Guide

## Why routeTree.gen.ts might not update automatically

Based on research, here are the common reasons and solutions:

### 1. **Development Server Must Be Running**
   - The route tree is regenerated by the Vite plugin during development
   - **Solution**: Always ensure `npm run dev` is running when making route changes

### 2. **Plugin Configuration**
   - Sometimes the plugin needs explicit paths to correctly watch files
   - **Solution**: Added explicit `routesDirectory` and `generatedRouteTree` in `vite.config.ts`

### 3. **File System Watcher Limitations**
   - macOS/Linux have limits on file watchers (especially on large projects)
   - **Solution**: If issues persist, you may need to increase file watcher limits:
     ```bash
     # macOS (check current limit)
     sysctl fs.inotify.max_user_watches
     
     # Or use a tool like watchman
     ```

### 4. **Editor/IDE Interference**
   - Some editors (like WebStorm) can interfere with file watching
   - **Solution**: Try restarting the dev server after making route changes

### 5. **Manual Regeneration (Fallback)**
   If automatic generation fails, you can manually regenerate:
   ```bash
   npx @tanstack/router-cli generate
   ```
   Note: Running this command twice sometimes resolves issues.

### 6. **Plugin Order Matters**
   - The `tanstackRouter` plugin should be placed before `viteReact()` in the plugins array
   - **Current setup**: ✅ Correct order

## Current Configuration

The `vite.config.ts` now includes:

- Explicit `generatedRouteTree: './src/routeTree.gen.ts'`

## Testing

1. Start dev server: `npm run dev`
2. Create/modify a route file in `src/routes/`
3. Check if `src/routeTree.gen.ts` updates automatically
4. If not, restart the dev server

## Additional Notes

- The `.tanstack` directory in `.gitignore` is normal (plugin cache)
- The `routeTree.gen.ts` file should NOT be manually edited (it's auto-generated)
- If you delete route files, the route tree should update automatically when the dev server is running
